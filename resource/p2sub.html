Project 2: Submission Guidelines

 

  1. At least one member from each group should email the TA (venkates@cse) and provide the information about the group members.
  2. Your PFS package should consist of the following components
    1. A pfs.h file which lists all the interfaces exported by the package. Make sure that you also list all the include files (like types.h, string.h, etc.) in this file.
    2. The source code for all the components
    3. A makefile to easily compile your library with a test file.
    4. Any other files that are part of your project.
  3. Put all the files in one folder. You can name the folder using the CSE user ids of the group members. For example, if the CSE unix ids of the group members are venkates and anand, then the name of your directory should be either venkates-anand or anand-venkates.
  4. Tar the complete folder and give your tar file the same name as the original folder (i.e., venkates-anand.tar or anand-venkates.tar)
  5. Submit your files online at the angel course website at http://www.angel.psu.edu. Once you log on using your Penn State access id, you should see an drop box for the “Project 2 Source Codes”. Upload your tar file in that folder. This is you primary submission method. Your project will not be graded if you do not submit your code online before the deadline.
  6. You should also email your tar files to the TA (venkates@cse)

 

Important Notes

  1. The deadline for your submission is midnight of December 12, 2005. The drop box will be disabled after midnight.
  2. You have to submit a project report which you should (preferably) leave in TA’s mailbox. If you do have access to the mailroom, then you can hand it over to the TA in his office (337 IST). It is important that you clearly explain your design in the project report and also provide details about the implementation highlighting any assumption you may have made.
  3. You will need to set up an appointment with the TA for demonstrating the project. A separate email would be sent out at a later date asking you to sign-up for the demo
