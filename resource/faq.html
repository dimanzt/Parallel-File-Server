FAQs

 

1. What are the semantics of the pfs_read() and pfs_write() APIs?

Here is a brief description of the arguments of pfs_read() and pfs_write() function calls:

- pfs_read() – Attempt to read nbyte bytes into the buffer pointed to by buf, from the file associated with the open file descriptor filedes, starting at the position given by offset. Upon successful completion, pfs_read() returns a non-negative integer indicating the number of bytes actually read.

- pfs_write() – Attempts to write nbyte bytes from the buffer pointed to by buf to the file associated with the open file descriptor filedes staring at a position given by offset. Upon successful completion, pfs_write() returns the number of bytes actually written to the file.

Both the functions return -1 in case of an error.

 

Note that, the semantics of pfs_read() and pfs_write() are similar to the pread() and pwrite() functions in unix.

 

2. What happens when a client gets a revoke request for tokens that are currently being used?

Consider the following scenario. Client C1 currently holds tokens in the range [o1, o2]. Now suppose client C2 requests for tokens which also fall somewhere in the same range [o1, o2]. The metadata manager will send a revoke request to C1 for the token range requested by client C2. Note that C1 will relinquish the tokens (or a part of it) only if it has finished its operations corresponding the token range. Otherwise, C2 needs to be blocked until it gets its set of requested tokens.

 

3. What are the semantics of pfs_create()?

The pfs_create() attempts to create an empty file with the name specified by filename. The return value indicates whether the operation was a success or not. pfs_create() does not return an open file descriptor. The client needs to explicitly open the file using pfs_open() function call after creating the file.

 

4. What are the different modes in which a file can be opened?

For this project, the files will be opened in either read mode (indicated by ‘r’) or write mode (indicated by ‘w’). That is to say, the write mode is equivalent to the read-write mode of unix file system.
